# Milestone Checklist

Status: v0.1. Use this checklist for every milestone before claiming completion. The governing ritual is in [`./audit-rituals.md`](./audit-rituals.md); the limb definitions are in [`../foundation/adapted-8-limbs.md`](../foundation/adapted-8-limbs.md).

## Pre-milestone (before starting)

- [ ] The milestone is named in one sentence — its single intention.
- [ ] The relevant section of [`../foundation/commonunity-architecture-v0.2.md`](../foundation/commonunity-architecture-v0.2.md) is identified.
- [ ] The 8 limbs have been walked **as a decision frame** (each returns at least *"not yet, but the path is clear"*).
- [ ] Acceptance criteria are written down.
- [ ] If the milestone touches Compass / Cipher / Living Profile / cOMmons public projection, the sovereignty boundaries are explicitly considered.

## During the work

- [ ] No broad audits launched unless strictly necessary (credit-conscious).
- [ ] Implementation is small, targeted, and acceptance-checked.
- [ ] Any new logic that could be shared is built on (or extracted into) the canonical engine — no inline duplicates of Om Cipher logic.
- [ ] No secrets, tokens, or keys committed.
- [ ] Visual changes adhere to warm dark-field discipline (no neon, no arbitrary color).

## Pre-completion (integrity audit)

- [ ] Architecture brief re-read.
- [ ] Philosophical principles re-read.
- [ ] All 8 limbs walked **as an audit frame** and recorded using [`./integrity-review-template.md`](./integrity-review-template.md).
- [ ] All ⚠️ have a named owner and follow-up.
- [ ] All ❌ are resolved (or the milestone is reshaped and re-audited).

## Cross-surface coherence

- [ ] Om Cipher render is identical across Studio threshold, Studio Living Profile, Personal Home, cOMmons cover, and cOMmons Profile (modulo `full` vs. `node` render mode).
- [ ] No surface displays member data the member has not consented to project.
- [ ] Resonance — not directory browsing — remains the default cOMmons interaction.
- [ ] Self-serve sign-up has not been introduced anywhere as the primary entry path.

## Deployment

- [ ] Live URLs and any DNS / SMTP / domain changes are reflected in [`../architecture/deployment-model.md`](../architecture/deployment-model.md).
- [ ] Recent important commits, service IDs, and current state are reflected in [`../handoffs/next-thread-handoff.md`](../handoffs/next-thread-handoff.md).
- [ ] CORS surfaces include all canonical and Railway domains.

## Closing

- [ ] One-line entry added to [`./decision-log.md`](./decision-log.md).
- [ ] Open work items are tracked in the handoff doc rather than vanishing into chat.
- [ ] The next milestone is named — or the system is explicitly idle.
