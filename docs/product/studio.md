# Studio

Status: v0.1 stub. Canonical source: [`../foundation/commonunity-architecture-v0.2.md`](../foundation/commonunity-architecture-v0.2.md), section *"Studio"*.

## What Studio is

Studio is the member's **higher-octave toolkit and living workspace**. It is downstream of Compass, Om Cipher, and Living Profile — not their source.

Studio is where the person works with their new baseline. It provides creative, healing, productive, and innovative tools that help the member manifest Unity in practice.

## What Studio is not

- Studio is **not** where Om Cipher or Living Profile are generated. Those come from Compass.
- Studio is **not** a self-serve sign-up funnel.
- Studio is **not** a feed or social surface.

## Tools (current and planned)

Possible Studio tools per the architecture brief:

- Tuner and frequency tools
- Om Cipher refinement and export tools
- AI innovation and business canvas tools
- 8 limbs of Yoga and practice design tools
- Attunement tools
- Personal homepage builder
- Creative project tools
- Healing, course, retreat, and service-development tools

Currently deployed surfaces include Studio rooms, the threshold/torus, Tuner module, practice modules (`ashtanga-*`, `surya-namaskar`, `hatha-*`), and the Website Preview path (Personal Home builder).

## Threshold

Entry to Studio is mediated by the torus threshold — the visible form of the OM Field passage. See [`../foundation/om-field.md`](../foundation/om-field.md).

## Business model touchpoints

- Minimal included toolkit with membership.
- Advanced tools as paid upgrades.
- Courses, retreats, events, and services as further paid layers.
