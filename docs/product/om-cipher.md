# Om Cipher — Product Surface

Status: v0.1 stub. For foundational framing see [`../foundation/om-cipher.md`](../foundation/om-cipher.md). For full input contract and visual mappings see [`../foundation/commonunity-architecture-v0.2.md`](../foundation/commonunity-architecture-v0.2.md).

This document covers Om Cipher **as a product layer** — where it appears for the member, what they do with it, and how it is rendered across surfaces.

## Where the member encounters Om Cipher

| Surface | Mode | Role |
| --- | --- | --- |
| End of Compass | `full` | Initiation seal; first time the member sees their cipher |
| Studio threshold (torus) | `full` / animated | Access ritual; opens Studio rooms |
| Studio Living Profile | `full` | Member's source pattern, displayed alongside lived expression |
| Personal Home Page | `full` | The face the member shows the world |
| cOMmons cover / node | `node` | Simplified boundary/color render in field constellation |
| cOMmons Profile | `full` | Public projection of the cipher |
| Future `omcipher.com` | `full`, exportable | Standalone product (lighter inputs) |
| Future iOS app | `full`, animated, AR | Sigil as breathing personal symbol |

## Member actions on the cipher

- **View** in any of the surfaces above.
- **Refresh frequency signature** by a new Bhramari capture (updates color and cymatic core, not the geometric scaffolding).
- **Export** (planned): high-resolution SVG/PNG, print-ready PDF, animated version, AR object.
- **Refine** through Studio tools (e.g. updated Gene Keys data, name handle adjustments).

## What the member does NOT do

- The member does **not** design their cipher. It is generated, not chosen.
- The member cannot make the cipher "prettier" by ornament. Visual quality is a function of input quality and engine quality.
- The cipher is not used as a security primitive in v1.

## Product expansion paths

Three product layers, in order of richness:

1. **Inside CommonUnity** — richest version, full inputs from Compass.
2. **Standalone web app (`omcipher.com`)** — lighter, self-contained from name, birthdate, frequency/note, optional Gene Keys.
3. **iOS app** — sigil-on-phone product. Bar is high: the sigil must be beautiful enough to consider as a tattoo.

## Business model touchpoints

- High-resolution SVG/PNG downloads (paid)
- Print-on-demand products (prints, stickers, jewelry)
- Tattoo artist partnerships
- Premium animation and AR
- "Done-with-you" cipher refinement services

See [`commonunity-architecture-v0.2.md`](../foundation/commonunity-architecture-v0.2.md), section *"Business model"*.

## Immediate product decisions (carried forward)

From the architecture brief:

1. The current "sigil" should be renamed and reframed as Om Cipher.
2. The current OM badge visual is v0 and should be replaced by Om Cipher v1 field geometry.
3. Om should remain the default seed syllable, but it should not dominate the visual.
4. Om Cipher and Living Profile are Compass artifacts, not Studio-made artifacts.
5. Studio presents them as source/orientation; cOMmons consumes their public expression.
