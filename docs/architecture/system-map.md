# System Map

Status: v0.1. Canonical source: [`../foundation/commonunity-architecture-v0.2.md`](../foundation/commonunity-architecture-v0.2.md).

This document shows how CommonUnity's five primary layers fit together and how a member moves through them.

## Member journey

```text
Invitation
  → facilitated Compass process
  → Om Cipher + Living Profile
  → Studio
  → Personal Home Page
  → cOMmons / Field
```

The sequence is not arbitrary. Each step depends on the prior one. Members do not enter through a self-serve software funnel; they enter through an invitation-led, human-facilitated Compass process.

## Cross-cutting: the OM Field threshold

The torus threshold is the access ritual that mediates movement between layers (especially into Studio, and later into cOMmons). It is not a separate layer but the **passage** between layers. See [`../foundation/om-field.md`](../foundation/om-field.md).

## Where the canonical Om Cipher engine is invoked

```text
canonical Om Cipher engine
  → Studio preview / threshold
  → cOMmons cover / node
  → cOMmons Profile / Living Profile public expression
  → future standalone omcipher.com product
```

Visual divergence between these surfaces is a bug. See [`../foundation/om-cipher.md`](../foundation/om-cipher.md) and the architecture brief for the input contract.

## Boundaries

| Layer | Belongs to layer | Does NOT belong to layer |
| --- | --- | --- |
| Compass | Onboarding, intake, facilitation, source-data capture | Tooling, public expression |
| Om Cipher | Source pattern, deterministic geometry, identity token | Lived projects, narrative |
| Living Profile | Lived expression, evolving meaning | Fixed source attributes |
| Studio | Tools that work *with* Cipher + Profile | Generating Cipher or Profile from scratch |
| Personal Home | Member's chosen public refraction | Private workspace |
| cOMmons | Relational resonance | Browse-style directory by default |
